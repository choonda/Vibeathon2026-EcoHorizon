# Module 5: Post-Trip Dashboard & Carbon Ledger

## Setup & Data Layer
- [ ] Create the `trips` table in Supabase (`id`, `user_id`, `distance_km`, `eco_score`, `fuel_cost_rm`, `carbon_saved_kg`).
- [ ] Create the `TripModel` Dart data class.
- [ ] Create the abstract `TripRepository` interface.
- [ ] Implement `SupabaseTripRepo` to push the final JSON payload to the cloud.

## State Management (Riverpod)
- [ ] Create the `LedgerController` (`StateNotifier`).
- [ ] Implement a translator function to map saved carbon float values (kg CO2) into concrete UI strings (e.g., "1 Tree Planted in Johor Bahru").
- [ ] Implement logic to automatically update the user's `petrol_points_balance` upon a successful trip upload.

## Presentation Layer (UI)
- [ ] Build the `TripSummaryScreen` to display the final Eco Score, total distance, and financial breakdown (RM lost/saved).
- [ ] Build the `CarbonLedgerDashboard` widget (Main App Home).
- [ ] Use `fl_chart` to render a historical trend graph of the user's past trips.
- [ ] Bind the dashboard text blocks to display the concrete environmental impact strings generated by the `LedgerController`.
